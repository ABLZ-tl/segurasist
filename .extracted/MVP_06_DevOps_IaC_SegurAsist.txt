1. Resumen DevOps
La plataforma SegurAsist se opera 100% como código (IaC + GitOps) con cero cambios manuales en consola AWS para recursos productivos. Todo lo que no esté en Git no existe. La estrategia descansa en cuatro pilares: AWS Organizations + cuentas separadas; Terraform 1.7 con remote state versionado; GitHub Actions para CI/CD; observabilidad CloudWatch + GuardDuty + Security Hub centralizada en una cuenta security.

2.1 Service Control Policies (SCPs) clave
	•	Bloquear región fuera de us-east-1, us-east-2 (excepto IAM y CloudFront).
	•	Bloquear deshabilitación de GuardDuty, Security Hub, Config, CloudTrail.
	•	Bloquear creación de usuarios IAM (forzar IAM Identity Center).
	•	Bloquear creación de buckets sin cifrado y sin Block Public Access.
	•	Bloquear creación de Security Groups con 0.0.0.0/0 hacia puertos internos (RDS 5432, Redis 6379).
	•	Bloquear borrado de S3 Object Lock objects (modo Compliance).

4.2 Remote state
Bucket S3 segurasist-tfstate-{env} con versioning, SSE-KMS y Block Public Access. Lock con DynamoDB tabla terraform-lock-{env}. Cada ambiente con su backend separado para aislar blast-radius.
4.3 Provider y convenciones
terraform {
  required_version = ">= 1.7.0, < 2.0.0"
  required_providers {
    aws = { source = "hashicorp/aws", version = "~> 5.40" }
  }
  backend "s3" {
    bucket         = "segurasist-tfstate-prod"
    key            = "main.tfstate"
    region         = "us-east-1"
    dynamodb_table = "terraform-lock-prod"
    encrypt        = true
    kms_key_id     = "alias/segurasist-tfstate"
  }
}

provider "aws" {
  region = "us-east-1"
  default_tags {
    tags = {
      Project   = "SegurAsist"
      Env       = var.environment
      Owner     = "platform"
      ManagedBy = "Terraform"
      CostCtr   = "MAC-MVP"
    }
  }
}
4.4 Convenciones de naming
	•	Recursos: segurasist-{env}-{servicio}-{detalle} (e.g., segurasist-prod-rds-main).
	•	Tags obligatorios: Project, Env, Owner, ManagedBy, CostCtr.
	•	Variables: snake_case, defaults seguros (cifrado on, public off).
	•	Módulos: con README.md, examples/, tests/ (Terratest opcional fase 2).

5.4 Módulo WAF
AWS Managed Rules: AWSManagedRulesCommonRuleSet, AWSManagedRulesKnownBadInputsRuleSet, AWSManagedRulesSQLiRuleSet, AWSManagedRulesAmazonIpReputationList, AWSManagedRulesAnonymousIpList. 6.2 Workflow API (extracto)
name: api-ci
on:
  pull_request:
  push:
    branches: [main]
    tags: ['v*']

permissions:
  id-token: write   # OIDC a AWS sin claves
  contents: read

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: npm }
      - run: npm ci
      - run: npm run lint
      - run: npm run typecheck

  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env: { POSTGRES_PASSWORD: test, POSTGRES_DB: test }
        ports: ['5432:5432']
        options: --health-cmd pg_isready --health-interval 5s
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: npm }
      - run: npm ci
      - run: npx prisma migrate deploy
      - run: npm run test:unit
      - run: npm run test:integration
      - run: npm run test:cross-tenant   # GATE crítico

  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npx semgrep --config p/owasp-top-ten --error
      - run: npm audit --audit-level=high
      - uses: github/codeql-action/init@v3
        with: { languages: javascript-typescript }
      - uses: github/codeql-action/analyze@v3

  build-image:
    needs: [lint, test, security]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::${{ secrets.AWS_ACCOUNT_PROD }}:role/github-actions-api
          aws-region: us-east-1
      - uses: aws-actions/amazon-ecr-login@v2
      - run: |
          docker build -t segurasist-api:${{ github.sha }} .
          docker tag  segurasist-api:${{ github.sha }} ${{ env.ECR }}/segurasist-api:${{ github.sha }}
          docker push ${{ env.ECR }}/segurasist-api:${{ github.sha }}
      - name: Trivy scan
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: ${{ env.ECR }}/segurasist-api:${{ github.sha }}
          severity: HIGH,CRITICAL
          exit-code: 1

  deploy-staging:
    if: github.ref == 'refs/heads/main'
    needs: build-image
    runs-on: ubuntu-latest
    environment: staging
    steps:
      - uses: actions/checkout@v4
      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::${{ secrets.AWS_ACCOUNT_STAGING }}:role/github-actions-deploy
          aws-region: us-east-1
      - run: aws apprunner start-deployment --service-arn $STAGING_SERVICE_ARN
      - run: ./scripts/wait-for-healthy.sh staging
      - name: DAST OWASP ZAP
        run: ./scripts/run-zap.sh https://api.staging.segurasist.mx

8. Observabilidad y on-call
8.1 Stack
	•	Logs: CloudWatch Logs por servicio (App Runner, cada Lambda, RDS errors).
	•	Métricas: CloudWatch Metrics nativas + EMF custom desde la API.

11. Gestión de costos
Coincide con el Stack Definitivo: ~4,730 MXN/mes año 1, ~11,240 MXN/mes año 3. DevOps añade alarmas y disciplina:
	•	AWS Budgets: 70%/90%/100% del mensual con alerta a Slack #segurasist-cost.
	•	Cost Anomaly Detection con umbral USD 50/día.
	•	Etiquetado obligatorio Project/Env/CostCtr; reportes desglosados mensual.
	•	Reservas y Savings Plans evaluables en mes 6 (no antes; necesitamos baseline).
	•	Lifecycle policies S3 obligatorias para todos los buckets.
	•	Lambda concurrent execution caps por función para evitar runaway.
