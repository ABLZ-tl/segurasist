## Patrones convergentes (cross-cutting reforzados)

1. **MessageDeduplicationId en colas standard** (A3-03 + A5-01 + A9-09 + nuevo A3v2-01 + A3v2-08): **4 reportes lo confirman**. Es el bug operacional #1 del repo. Decisión arquitectónica urgente Sprint 4.
2. **`SQS_QUEUE_INSUREDS_CREATION` ausente** (A3-04 + A9-04 + A9-LOW + nuevo A3v2-09): drift entre `localstack-bootstrap.sh` (crea 5), Terraform (crea 4), env.schema (declara 4), workers (fabrican la 5ª por replace). 4 sitios para mantener; 0 fuentes de verdad.
3. **Tests it.todo en cross-tenant HTTP** (A2-34 + A10-03 + nuevo A3v2-04): batches tiene 3 todos confirmados; total repo 23 todos. Sin estos, RLS sólo se valida BD-side.
4. **Counter race + state machine** (A3-02 + A3-08 + nuevo A3v2-01 + A3v2-03): el flujo `confirm → N mensajes → batch.completed` tiene 4 race conditions documentadas. Necesita rediseño con `expectedQueued`/`processedCount` columnas + UPDATE atómico con CAS.
5. **A4 hash contract roto + A3 emite el evento que A4 consume** (A4-01 + A3v2-02): si A3 re-entrega `insured.created` (porque dedupeId se ignora) y A4 genera dos PDFs con hashes random distintos, **el verify endpoint público falla determinísticamente para uno de los dos**. Cross-cutting Critical.

## Recommendations Sprint 4 (top 5 — actualizado)

1. **[Critical] Fix A3-01 + A3-02 + nuevos A3v2-01/A3v2-03** con tests reproductores en mismo PR. Pre-computar dups antes del loop de chunks; agregar columnas `processed_rows`/`queued_count` al schema; convertir `bumpBatchCounters` a UPDATE...RETURNING con CAS para emitir `batch.completed` exactly-once.
2. **[Critical-architectural] Decisión FIFO vs DB-idempotency** (A3-03 + A5-01 + A9-09 + A3v2-08): recomendación firme = **DB-idempotency** (eliminar `MessageDeduplicationId` del SqsService; idempotencia por UNIQUE constraints + status checks). Costo: cambio puntual `sqs.service.ts:26`. Beneficio: cierra 4 issues de 4 reportes distintos. Si la decisión es FIFO, requiere extender el método con `messageGroupId`, renombrar 5 colas, agregar Terraform, y bootstrap LocalStack — esfuerzo M en lugar de XS.
3. **[High] Cerrar `SQS_QUEUE_INSUREDS_CREATION` end-to-end** (A3-04 + A3v2-09 + A9-04 + A10): env.schema + .env.example + Terraform map `local.queues` + workers leen del env (no string-replace). Crear `docs/audit/QUEUES.md` (nuevo A3v2-10) como fuente única.
4. **[High] Tests cross-tenant HTTP batches** (A3v2-04 + A2-34 + A10-03): convertir los 3 it.todo + 3 nuevos (`preview`, `errors`, `errors.xlsx`) a tests reales. Esfuerzo S (~0.5 días); ROI: cubre el gap más visible del audit Sprint 3.
5. **[Cross-cutting] Coordinar A3↔A4 para `batch.completed` exactly-once + `insured.created` idempotente** (A3v2-02): hoy depende de dedupe SQS que no funciona en standard. Mover a UNIQUE `(tenantId, insuredId, version)` en `certificates` + INSERT ON CONFLICT antes del Puppeteer render. Sprint 4 PR cross-team.
