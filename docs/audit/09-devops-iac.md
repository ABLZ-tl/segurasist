# Audit Report — DevOps + Terraform + IaC + scripts (A9)

## Summary (≤10 líneas)

Auditoría READ-ONLY del estado IaC al cierre Sprint 3. Cobertura: 16 módulos Terraform, 3 envs (`dev/staging/prod`), 4 stacks `global/` (organization, security, route53, iam-github-oidc), 3 workflows infra + 1 CI raíz, 7 scripts shell de bootstrap/operación, 12 runbooks, 3 docs de seguridad, ZAP rules + script. La estructura modular y los defaults seguros (cifrado on, public-access off, deletion_protection on, Object Lock COMPLIANCE 24m, RLS via roles DB) son sólidos. Hay **3 issues Critical** que bloquean el pipeline real (api-DAST contra `/v1/openapi.json` que no está montado, `github-actions-tf-plan-{staging,prod}` no existen pese a estar referenciados, falta CW alarms instanciados ⇒ runbooks ciegos). Los workflows DAST/secscan están listos pero dependen de gating roles externos. Drift severo: la cola `insureds-creation` que usan workers no se provisiona en Terraform. Trivy / build-image no está en CI (gap S2 conocido).

## Strengths (qué está bien hecho)

- **Higiene Terraform**: 100% de los módulos tienen el quinteto `main / variables / outputs / versions / README`. Default tags consistentes en los 3 envs (`Project=SegurAsist, Env, Owner=platform, ManagedBy=Terraform, CostCtr=MAC-MVP`). Versionado `~> 5.40` AWS y `>= 1.7.0, < 2.0.0` Terraform en `versions.tf` por módulo.
- **Patrones modernos**: uso correcto de `for_each` con maps en SQS, KMS, lifecycle rules; `aws_vpc_security_group_*_rule` (single-rule) en lugar de los antiguos `ingress/egress` blocks; `configuration_aliases` para cross-region en `rds-postgres` (`aws.replica`) y `route53/providers.tf` (`aws.us_east_1`).
- **Defaults seguros endurecidos**: `s3-bucket` aplica TLS-only deny policy + `BucketOwnerEnforced` + `block_public_access` por default; `cognito-user-pool` arranca con `mfa_configuration=ON` para admin pool, `software_token_mfa_configuration` y account recovery via `verified_email`; `kms-key` con `enable_key_rotation` por default y policy bien estructurada (root + service principals + additional principals).
- **WAF**: 5 AWS Managed Rules (Common, KnownBadInputs, SQLi, AmazonIpReputation, AnonymousIpList) + rate-based; `AnonymousIpList` correctamente en COUNT (validado contra hospitales con VPN corporativa); redacción `authorization`/`cookie` en logs antes de Firehose. Comentarios explican el tradeoff y la migración programada a BLOCK.
- **Org-wide**: SCPs (region-restrict, deny-disable-guardduty/SH, deny-iam-user, deny-s3-public/unenc, deny-object-lock-delete) + GuardDuty + Security Hub (CIS v2 + AWS Foundational) + Config aggregator + Inspector v2 + CloudTrail org-wide en bucket Object Lock COMPLIANCE en log-archive account. Patrón maduro.
- **Scripts idempotentes**: `apply-rls.sh` chequea roles tras el apply; `localstack-bootstrap.sh` head-bucket antes de create; `cognito-local-bootstrap.sh` `ensure_pool/ensure_client/ensure_user` con verificación previa; `backup.sh` calcula sha256 y verifica integridad en `restore.sh` antes del WIPE; `seed-bulk-insureds.sh` usa `ON CONFLICT (tenant_id, curp) DO NOTHING`.
- **CI raíz** con `paths-filter`, gating job `ci-success`, `concurrency.cancel-in-progress=true`, separación lint/unit/sec/e2e/dast por sub-proyecto, dependency-review con licencia denylist (GPL-3.0/AGPL-3.0).

## Recommendations Sprint 4 (top 5)

1. **Desbloquear el pipeline CI/CD real (Critical)**: completar `SwaggerModule` en `main.ts`, crear `tf_plan_{staging,prod}` roles + agregar job `api-build-image` con Trivy. Sin esos 3, el gate `ci-success` no puede pasar verde y todo el flow GitHub Actions queda esperando.
2. **Wirear CloudWatch alarms (Critical/High)**: crear `envs/{env}/alarms.tf` con SNS topic dedicated + 7-10 alarmas core (App Runner 5xx, RDS CPU/conn, SQS DLQ depth, WAF blocked spike, SES bounce/complaint, Lambda errors, cross-tenant audit log filter). Mapear a runbooks RB-00x existentes; cerrar gap H-04 y dar señal real al on-call.
3. **Resolver SQS FIFO vs idempotencia DB**: definir si `reports-queue` y `insureds-creation-queue` necesitan FIFO. Recomendación: NO FIFO (idempotencia en DB ya implementada via row-state check); por lo tanto, eliminar `MessageDeduplicationId` de `SqsService.sendMessage`. Agregar `insureds-creation` al map `local.queues` y al `env.schema.ts/.env.example`.
4. **Completar runbooks P1 + IRP**: RB-001, RB-005, RB-008, RB-010 + IRP roles/severities con contenido real (no TBD). Tabletop exercise mínimo en Sprint 4 para validar.
5. **Endurecer permisos + SCPs**: reemplazar `PowerUserAccess` y `AdministratorAccess` en deploy/tf-apply roles por inline policies derivadas de los recursos reales (script `terraform-policy-from-state`). Quitar excepción `AWSReservedSSO_AdminFullAccess_*` del SCP `deny-s3-object-lock-delete`. Agregar segundo thumbprint GitHub OIDC.
